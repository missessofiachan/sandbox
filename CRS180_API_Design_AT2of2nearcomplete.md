|<p>This assessment task requires learners to plan and build a Restful API that saves and stores data in a database. (Minimum 2 paragraphs)</p>|<p>*Securing a REST API with JWT and Bcrypt*</p><p>*One of the most effective and widely adopted methods for securing a REST API is the use of JWT (JSON Web Tokens) for authentication and authorization, combined with bcrypt for password hashing. JWT is a compact, URL-safe means of representing claims to be transferred between two parties. In a typical workflow, when a user logs in with their credentials, the server verifies the credentials (with bcrypt-hashed password comparison) and, if valid, issues a signed JWT. This token is then sent to the client and must be included in the header of subsequent API requests. The server validates the token on each request, ensuring that only authenticated users can access protected endpoints. This stateless approach scales well and is language-agnostic, making it ideal for modern web APIs.*</p><p></p><p>*In this project, Express.js is used as the framework, with the jsonwebtoken library for JWT handling and bcrypt for password encryption. Sensitive endpoints (such as user management, product creation, and order processing) are protected by middleware that checks for a valid JWT in the request headers. Role-based access control is also implemented, allowing only users with the "admin" role to access certain routes. Storing secrets and sensitive configuration in environment variables (using a .env file) further enhances security by preventing accidental exposure of credentials in source code. This layered approach—combining strong password hashing, token-based authentication, and role-based authorization—ensures that the API is robustly protected against common threats such as credential theft, unauthorized access, and token forgery.*</p><p></p><p>*Frameworks and Techniques Used:*</p><p></p><p>*Express.js (API framework)*</p><p>*jsonwebtoken (JWT authentication)*</p><p>*bcrypt (password hashing)*</p><p>*dotenv (environment variable management)*</p><p>*Custom middleware for authentication and role-based authorization*</p><p>*This approach aligns with industry best practices.*</p>||
